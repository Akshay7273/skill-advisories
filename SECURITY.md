# Security Policy

## Reporting a malicious skill

Open a ["Report a malicious or vulnerable skill"](https://github.com/Akshay7273/skill-advisories/issues/new/choose) issue with links to published evidence.

For sensitive reports that should not be public yet (e.g. an unpublished finding or an ongoing takedown), email **akshaykumar7273@gmail.com** instead.

## Reporting a vulnerability in this project's code

Email **akshaykumar7273@gmail.com**. Please do not open a public issue for unpatched vulnerabilities in the validator, CLI, or Action.

## Disputes and withdrawals

If you believe an advisory is incorrect — especially if you are the author of the flagged artifact — open a **"Dispute an advisory"** issue. Disputes are reviewed against the cited references.

If an advisory is found to be wrong, it is **withdrawn**: the record stays in the repository with a `withdrawn` timestamp for auditability, but it is removed from the lookup index, so downstream tools immediately stop flagging the artifact.
